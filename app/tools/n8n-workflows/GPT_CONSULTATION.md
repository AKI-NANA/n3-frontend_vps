# GPT相談用：n8nワークフローUI設計について

## 背景

N3という物販管理システムを開発しています。このシステムは170以上のn8nワークフロー（自動化処理）を持っており、それらを管理・可視化するためのUI画面を作成しようとしています。

現在、基本的な一覧表示はできましたが、UIレイアウトが崩れており、また表示すべき情報が不足しています。

## 現状の問題

### 1. UIレイアウトの崩れ
- 統計カード（総数、アクティブ、テスト中、非推奨）が横4列に並ぶべきなのに縦に並んでいる
- 検索バーとボタンの配置がおかしい
- カード内のコンテンツが適切に整列していない

### 2. 表示情報の不足

**現在表示している情報：**
- ワークフロー名（日本語・英語）
- ステータス（稼働中/テスト中/非推奨）
- カテゴリ（出品/在庫/AIなど）
- バージョン番号
- 説明文
- Webhook Path（APIエンドポイント）

**足りていない重要な情報：**
- ローカルJSONファイル名（例：`【出品】01_ローカル-eBay出品処理-完成版v6修正_V7.json`）
- n8nの実ワークフローIDとの紐付け状況（連携できているか、できていないか）
- UIボタンとの連携状況（システム内のどのページのどのボタンから実行できるか）
- ワークフローのタイプ（通常のワークフロー/AIエージェント/Python連携/Cron自動実行）
- **高校生でもわかる簡易説明**（何をするツール？いつ動く？何が起きる？）
- 依存関係（他のワークフローや外部サービスとの連携）

### 3. 機能未実装
- 「詳細」ボタン → 何も起こらない
- 「実行」ボタン → 何も起こらない
- 各ボタンをクリックしたときの挙動が未定義

## 参考：既存のN3ツールのデザイン

このシステムには既に7つの主要ツールがあり、それぞれ統一されたデザインシステムを使用しています：

- **editing-n3**（データ編集）：3カラムレイアウト、フィルタータブ、商品カード
- **listing-n3**（出品管理）：L2タブナビゲーション、統計カード、アクションボタン
- **operations-n3**（運用管理）：ステータスバッジ、詳細モーダル

これらと同じデザインシステム（N3コンポーネント）を使って統一感を出したいです。

## 技術的な制約

- フレームワーク：Next.js 15 + React
- デザインシステム：独自のN3コンポーネント（N3Flex, N3Grid, N3FilterTabなど）
- データソース：workflow-registry.ts（TypeScript配列、現在10件登録済み）
- 実際のワークフロー：170+件のJSONファイル（別途存在する可能性）

## 質問

このような「ワークフロー管理UI」を設計する場合、以下の点についてアドバイスをください：

### 1. 情報設計
高校生でも理解できるように、各ワークフローについて**どのような情報を、どのような順番で、どのように表示**すべきでしょうか？

特に以下の点が重要です：
- 「何をするツールか？いつ動くか？何が起きるか？」を分かりやすく説明する方法
- 技術的な詳細（Webhook Path, JSON filename）と分かりやすい説明のバランス
- 連携状況（UIボタンとの紐付け、n8nとの同期状態）の可視化方法

### 2. UIレイアウト
170件のワークフローを**効率的に閲覧・検索・フィルタリング**できるレイアウトとして、どのような構成が最適でしょうか？

考慮すべき点：
- 一覧性と詳細性のバランス
- カテゴリ（出品/在庫/AIなど16種類）とステータス（稼働中/テスト中/非推奨）での絞り込み
- 検索機能の配置
- カード型 vs テーブル型 vs その他

### 3. インタラクション設計
各ワークフローに対して、ユーザーができるべきアクション（ボタン）は何でしょうか？

候補：
- 詳細を見る（モーダルで表示？別ページ？）
- テスト実行（パラメータ入力が必要な場合は？）
- n8nダッシュボードで開く（外部リンク）
- JSON定義をダウンロード
- 編集・無効化・削除

### 4. 優先順位
限られた開発時間で最大の価値を提供するため、**どの機能から実装すべき**でしょうか？

現在考えている段階：
- フェーズ1：UIレイアウトの修正（即時）
- フェーズ2：情報追加（高校生向け説明、連携状況）
- フェーズ3：ボタン機能実装（詳細モーダル、実行機能）
- フェーズ4：データ統合（170+件の自動登録、n8n API連携）

## 参考になるUI例

以下のようなサービスのUIが参考になると考えていますが、他に良い例があれば教えてください：
- GitHub Actions（ワークフロー一覧、実行履歴）
- Zapier Dashboard（トリガー/アクション、連携状況）
